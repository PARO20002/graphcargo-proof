The Role of TAC Index

TAC is our Confirmation Layer — not the product

GraphCargo combines 15+ data sources into a causal model that predicts air cargo yield movements. TAC Index serves one specific role: market confirmation. It sits between macro-economic leading indicators and the actual yield outcome.

Why TAC Index is essential

Without TAC: We can detect macro shifts, but we can't confirm whether the air cargo market has reacted yet.
With TAC: BAI indices confirm the price signal. TAC Space confirms physically (actual flights vs. schedule). This turns a statistical hypothesis into a validated forecast.

Data Usage

What we do — and what we will never do

✅ What we do with TAC data

  • Classical statistics: Pearson correlation, lag analysis, seasonal decomposition, Granger causality testing
  • One signal among 15+: TAC is combined with OAG, FRED, FAO, GSCPI, WACD, Comtrade — never used in isolation
  • Derived composite scores: Corridor condition scores that blend multiple sources — no raw TAC values exposed
  • Full attribution: "Data by TAC Index" badge wherever TAC data contributes
  • Internal time-series storage: For rolling-window backtesting and historical pattern matching

🚫 What we will never do

  • No LLM training: TAC data is never used to train language models, create embeddings, or fine-tune generative AI
  • No competing index: We will never publish a rate benchmark or settlement product
  • No raw data exposure: End users cannot download, export, or reconstruct TAC raw data
  • No redistribution: TAC data is never resold, sublicensed, or shared with third parties
  • No scraping: We will use licensed API access only — no web scraping of TAC platforms
Technical Architecture

How TAC data flows through our system

TAC data enters as one input signal, gets correlated with 14 other sources, and leaves as a blended corridor score. At no point can raw TAC values be extracted from the output.

StepWhat happensTAC data state
INGEST TAC API delivers BAI, TAC Freight, and TAC Space data via licensed feed Raw values stored in internal database (encrypted, access-controlled)
CORRELATE Signal engine computes Pearson correlation with lag (1–12 weeks) against every lane × commodity pair from WFC/customer AWB data TAC values become one feature in a multi-source correlation matrix (163,000+ pairs)
GRAPH Statistically significant relationships are written to Neo4j knowledge graph as CONFIRMS or PREDICTS edges Only the relationship metadata (r, lag, direction) — not the raw TAC value
OUTPUT Customer sees: "Corridor EU→NAM: Yield rising, confidence 8.3/10, 4-week horizon" No raw TAC data visible. Only derived score + "Data by TAC Index" attribution
