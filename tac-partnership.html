Data Usage Commitments
What we do, and what we will never do.
We will
  • Classical statistics only -- Pearson correlation, lag analysis, seasonal decomposition, Granger causality tests
  • Blend with 14 other sources -- TAC is one signal among OAG, FRED, FAO, GSCPI, WACD, Comtrade
  • Output derived scores -- corridor confidence scores that no longer contain raw TAC values
  • Attribute fully -- "Data by TAC Index" wherever TAC contributes to a customer-facing output
  • Comply with Level 2 -- TAC Terms of Use and Data Policy framework
We will never
  • Train AI models -- no LLMs, no embeddings, no generative AI fine-tuning with TAC data
  • Publish a competing index -- no rate benchmark, no settlement product, no price publication
  • Expose raw data -- end users cannot see, download, export, or reconstruct underlying TAC values
  • Redistribute -- no resale, no sublicensing, no sharing with third parties
  • Scrape -- licensed API access only, no crawling of TAC platforms
Technical Architecture
How TAC data flows through our system.
TAC data enters as one input, gets correlated with 14 other sources, and leaves as a blended corridor score. Raw TAC values cannot be extracted from the output.
1
Ingest -- TAC API delivers BAI, TAC Freight, and TAC Space via licensed feed
Raw values stored encrypted, access-controlled, internal only
2
Correlate -- Signal engine tests lag correlation (1--12 weeks) against each lane and commodity
TAC becomes one feature in a 163,000-pair correlation matrix
3
Graph -- Significant relationships written to Neo4j as CONFIRMS or PREDICTS edges
Only metadata persists: r-value, lag, direction. Not the raw TAC value.
4
Output -- Customer sees: "EU to NAM: yield rising, confidence 8.3, 4-week horizon"
No raw TAC data visible. Attribution: "Data by TAC Index"
